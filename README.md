## サービス概要
仕事・趣味・転職後など、複数の人格（プロファイル）を切り替えて使えるツール群のうちのToDo管理アプリ。
プライベートでは組織・グループ、仕事では本業／副業・組織ごとにプロファイルを切り替えながら、一貫した操作と情報整理が可能。
「同じ自分がどんな環境でも使い続けられる」ことを目指した、新しい形のToDoアプリ。

## このサービスへの思い・作りたい理由
自分自身、仕事・学習・プライベートのタスクをそれぞれ異なるツールで管理しており、転職も経験しました。
しかし、ツールやアカウントを切り替えるたびに操作方法やUIが違い、ストレスを感じていました。
また、社外とのやり取りでは多岐にわたるコミュニケーションツールをすべてチェックしていく必要があります。
さらにプライベート／ジョブのカレンダー・ToDOも統合的に使うことができず、アカウントを切り替える必要があります。
加えて昨今は業務の一部分ずつを担うWebアプリケーションが乱立しており、しかしながらアプリケーション間の連携には限界があります。
この経験から、一貫したユーザーと切り替え可能なプロファイルの二重構造、かつ一つのサービス内にさまざまな機能があるツールが必要だと考えました。
今回はMVPとして、上記の①User／Profile／Organization構造と、②（様々な機能の中から）ToDo機能を用意したいと思います。

## データ構造の考え方
本サービスでは、主に3つのレイヤーを作ります。

- User：ログイン情報・本人確認情報など「個人」を識別する起点。
- Profile(Prof)：Userが持つ複数の社会的「人格」「役割」を表現（例：会社A／副業B／趣味コミュニティ／家庭私用）。
  User : Prof = 1 : N で、ユーザーは複数のProfを持ち、TaskはProfを介してUserに紐づきます。
  ログアウト・ログインをせずとも、Profを切り替えて作業したり、全Profを横断して管理できます。
- Organization(Org)：会社・学校・サークルなどの所属先。
  Prof : Org = N : N で、所属を表現します。

全てのProfを統合して管理もでき、むしろ二重管理を回避する構造を目指しています。
外部サービス（Google, Slack, Notionなど）との連携はMVPでは行わず、本アプリ内で完結する構造とします。

## ユーザー層について
- 仕事とプライベートを分けつつ一括管理したい人
- 副業・転職・個人開発など複数の役割を持つ人
- 組織を変えても一貫したアプリ操作をしたい人
この層は日常的に多岐に渡るツールを活用する必要があり、煩雑さを感じているユーザーが多いと考えました。

## サービスの利用イメージ
User登録後、本業・副業・プライベートなど複数のProfを作成します。
Profごと、または全Prof横断からタスクを登録・管理できます。
画面上部のドロップダウンなどから、ワンクリックでプロファイルを切り替えられます。
切り替えに応じてUIテーマ・タスク一覧が瞬時に変化します。
これにより、ユーザーはプロファイルごとの切り替え・一貫した操作・統合された情報を体感し、「その時の自分」に集中できる環境を得られます。

## ユーザーの獲得について
今回は職場向けToDoツールのみですが、ゆくゆくはチャット機能など他ツールをこのサービス内に追加します。
職場のツールを入り口にしつつ、他の入り口を多く作ります。
導入する会社が増えれば、人生を通して使えるプロダクトになります。
- 職場で使いやすいツールとして最初の指示を獲得する。
- SNS（X / Instagram）でユーザー-プロファイル構造を軸に発信。
- 複数のプロジェクトを抱えるユーザーに向けてポートフォリオとして公開。
- Qiitaなどで開発記事を投稿し、共感からユーザーを増やす導線を想定。

## サービスの差別化ポイント・推しポイント
Trello 複数アカウントのマージは不可。
Notion　複数ワークスペース・複数アカウントが切り替えできるが、通知・既定設定がアカウント依存。

本サービスではアカウントやワークスペースを切り替えるのではなく、Userを共通の軸にProfのみを切り替える・横断する設計にします。
使用者の一貫性と、社会的役割の多面性に対応します。
ゆくゆくは以下の機能を追加し、その全ての機能においてUser/Prof構造を適用し、機能間で円滑・自由な連携を目指します。

## 機能候補
MVPリリース時に実装予定
- ユーザー登録 / ログイン機能
- プロファイル作成・削除・切替
- User / Prof / Org の構造
- タスクのCRUD
- タスクの依頼
- タスクのアクティビティログ（作成・更新・完了）、コメント機能
- カレンダー表示
- Prof横断でのタスク管理
- 他機能でも横断できる設計（共通プリミティブの用意）
  他機能の実装は本リリースのさらに先の構想です。
  Profileのページ化・充実（自己紹介、ポートフォリオの公開、書類・作品などの作成・公開・保管、活動履歴の集計）
  Chat機能（他Userとルーム式のチャット機能）
  Social機能（内外への発信）

    - 想定している共通プリミティブ
      下記を共通のデータ構造として設計します。
      将来的にToDo以外の機能（Chat・Socialなど）が追加されても、同じ構造で横断的に扱うことを目指します。
        - Item：ユーザーが作るデータの共通形式（MVPでは主にTaskが該当、他Event・Post・Comment・Noteなど）
        - Activity：作成・更新・削除などの行動履歴
        - Comment：Itemに付与できるコメント機能、Itemの一つでもある
        - Reaction：ItemやCommentに付与できる「いいね」機能またはスタンプ機能
        - Tagging：タグ付け機能
        - Notification：通知機能
        - Share：Itemの共有機能、Itemの一つでもある
 

本リリースまでに追加したい機能
- タスクの期日・通知設定・繰り返し設定
- 他ユーザーとの連携（チームタスク、他ユーザーへのタスク委譲・依頼など）
- カレンダー表示の充実
- プロファイルのフォロー機能
- タグ付け機能
- PWA対応（モバイルでも利用可能）

## 使用する技術スタック
- フレームワーク	Ruby on Rails 7.x	Runteq標準。Hotwireで非同期操作も可能。
- データベース	PostgreSQL	各プロファイル・タスク管理に最適。
- フロントエンド	Tailwind CSS / Hotwire	シンプルな切替体験を実装。
- 認証	Devise	ユーザー管理を簡易に実装。
- デプロイ	Render	無料枠でMVP運用可能。
- テスト	RSpec	モデル・機能テストを実装予定。
- Linter	Rubocop	コード品質維持。
