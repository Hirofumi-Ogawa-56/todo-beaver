## サービス概要
仕事・趣味・転職後など、複数の人格（プロファイル）を切り替えて使えるツール群のうちのToDo管理アプリ。
プライベートでは組織・グループ、仕事では本業／副業・組織ごとにプロファイルを切り替えながら、一貫した操作と情報整理が可能。
「同じ自分がどんな環境でも使い続けられる」ことを目指した、新しい形のToDoアプリ。

## このサービスへの思い・作りたい理由
自分自身、仕事・学習・プライベートのタスクをそれぞれ異なるツールで管理しており、転職も経験しました。
しかし、ツールやアカウントを切り替えるたびに操作方法やUIが違い、ストレスを感じていました。
また、社外とのやり取りでは多岐にわたるコミュニケーションツールをすべてチェックしていく必要があります。
さらにプライベート／ジョブのカレンダー・ToDOも統合的に使うことができず、アカウントを切り替える必要があります。
この経験から、一貫したユーザーと切り替え可能なプロファイルの二重構造にすることで、さまざまなツールを円滑にかつ一貫して使えるツールが必要と考えました。
今回はMVPとして、上記の①User／Profile／Group構造と、②（様々なツールの中でも）ToDo機能を用意したいと思います。

## ユーザー層について
- 仕事とプライベートを分けつつ一括管理したい人
- 副業・転職・個人開発など複数の役割を持つ人
- 組織を変えても一貫したアプリ操作をしたい人
この層は、日常的にタスク管理を始めとする情報ツールが多岐に渡っている状況に煩雑さを感じているユーザーが多いと考えました。

## サービスの利用イメージ
ユーザー登録後、本業・副業・プライベートなど複数のプロファイルを作成します。
プロファイルごとにタスクを登録・管理できます。
画面上部のドロップダウンなどから、ワンクリックでプロファイルを切り替え。
切り替えに応じてUIテーマ・タスク一覧が瞬時に変化。
全てのプロファイルのToDoを一元管理することもできます。
この体験により、ユーザーはプロファイルごとの切り替え・一貫した操作・統合された情報を体感し、「その時の自分」に集中できる環境を得られます。

## ユーザーの獲得について
今回はToDoツールのみだが、ゆくゆくはチャット機能・SNS機能・ドキュメント機能・ショッピング機能などを広く追加し、入り口を多く作る。
ユーザーアカウントを作り、導入する会社が増えれば、人生を通して使えるプロダクトになる。
- SNS（X / Instagram）でユーザー-プロファイル構造を軸に発信。
- 複数のプロジェクトを抱えるユーザーに向けてポートフォリオとして公開。
- Qiitaなどで開発記事を投稿し、共感からユーザーを増やす導線を想定。

## サービスの差別化ポイント・推しポイント
Trello 複数アカウントのマージは不可。
Notion　複数ワークスペース・複数アカウントが切り替えできるが、通知・既定設定がアカウント依存。

ToDo-Beaverは、アカウントやワークスペースではなく、プロファイルのみを切り替える。
ユーザーとしては同じなので、一貫性・統合性が必要な場面で、ユーザー内の情報を網羅して操作できる。

## 機能候補
MVPリリース時に実装予定
- ユーザー登録 / ログイン機能
- プロファイル作成・削除・切替機能
- プロファイルごとのタスクCRUD
- タスクから、カレンダー上のイベントの自動生成
- タスク一覧の表示（プロファイルごと、全プロファイルの一括管理）
- 他ツールでも横断できる設計（CRUDの記録、コメント・リアクション機能、通知機能）

本リリースまでに追加したい機能
- タスクの期日・通知設定・繰り返し設定
- 他ユーザーとの連携（チームタスク、他ユーザーへのタスク委譲・依頼など）
- カレンダー表示の充実
- プロファイルのフォロー機能
- タグ付け機能
- PWA対応（モバイルでも利用可能）

## 使用する技術スタック
使用する技術スタック
- フレームワーク	Ruby on Rails 7.x	Runteq標準。Hotwireで非同期操作も可能。
- データベース	PostgreSQL	各プロファイル・タスク管理に最適。
- フロントエンド	Tailwind CSS / Hotwire	シンプルな切替体験を実装。
- 認証	Devise	ユーザー管理を簡易に実装。
- デプロイ	Render	無料枠でMVP運用可能。
- テスト	RSpec	モデル・機能テストを実装予定。
- Linter	Rubocop	コード品質維持。
